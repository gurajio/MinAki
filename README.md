# MinAki

# みんなの空き時間

# 概要
本アプリは，大人数で予定を合わせる際に，参加者の予定内容を共有せず，時間枠ごとの空き具合のみを集約して可視化するWeb日程調整ツールである。従来の日程調整サービスでは，幹事が候補日を事前に設定し，参加者がそれぞれ回答する形式が一般的であった。しかし，参加者が多い場合，候補日の設定や回答結果の確認が煩雑になり，最適な日時を見つけることが難しい。そこで本アプリでは，参加者の空き時間をカレンダー形式で収集し，各時間枠の参加可能人数をカラーグレードで表示する。これにより，個人の予定内容を非公開にしたまま，集まりやすい時間帯を直感的に把握できる。


# 目的

- **大人数で予定を合わせる際に、個人の予定内容を共有せず、時間枠ごとの空き具合だけを可視化することで、日程調整の負担を減らすこと**です。
- 従来の日程調整では、幹事が候補日を作成し、参加者がそれぞれ回答し、その結果を見て最適な日時を判断する必要があります。しかし、30人規模のように人数が多くなると、候補日の設定や回答確認が手間になり、全員の予定を把握することも難しくなります。
- そこで本サービスでは、参加者が自分の予定内容を公開しなくても、各時間枠に何人が参加可能かを集計し、色の濃淡で表示します。これにより、幹事はどの時間帯が最も集まりやすいかを直感的に把握できます。

# 解決したい課題

### 1. 大人数の日程調整が面倒

30人程度の予定を合わせる場合、LINEやメッセージで一人ずつ予定を聞くのはかなり手間です。

また、調整さんやLINEスケジュールのようなサービスでも、幹事が先に候補日を決める必要があります。

しかし、大人数の場合はそもそもどの日がよさそうか分からないことが多いです。

このアプリでは、先に候補日を絞り込むのではなく、広めの期間の中から集まりやすい時間を可視化できます。

### 2. GoogleカレンダーやTimeTreeは導入が重い

GoogleカレンダーやTimeTreeは、継続的に予定を共有する相手には便利です。

しかし、一時的なイベントや30人規模の集まりでは、全員を招待したり、友達登録したり、アプリを入れてもらったりする手間があります。

このアプリは、WebサイトのURLを共有するだけで参加できるため、一時的な大人数の日程調整に向いています。

### 3. 予定の中身は共有したくない

予定を合わせるために必要なのは、何をしているかではなく、その時間に参加できるかどうかです。

しかし、既存のカレンダー共有では、予定名や詳細を見られる不安があります。

このアプリでは、参加者の予定内容は共有せず、時間枠ごとの空き人数だけを表示します。

つまり、個人の予定内容を隠したまま、集団としての空き具合だけを共有できます。

### 4. 回答結果が見づらい

人数が多くなると、誰がどの日に参加できるかを一覧で見るだけでは判断しづらくなります。

特に30人規模では、○×の表を見るだけでも負担になります。

このアプリでは、時間枠ごとの参加可能人数をカラーグレードで表示します。

空いている人が多い時間ほど濃く表示されるため、候補時間を直感的に見つけられます。

# アプリのコンセプト

**予定の中身を見せずに、みんなが集まれそうな時間だけを見つけるWeb日程調整ツール。**

もう少し短くすると、

**誰が何をしているかではなく、いつ集まれそうかだけを見るカレンダー。**

さらにサービス紹介風にすると、

**Akiiroは、大人数の予定調整を色で見える化するWebカレンダーです。参加者の予定内容は非公開のまま、時間枠ごとの空き人数だけを集計し、集まりやすい時間を直感的に探せます。**

# 想定ユーザー

主な対象は、**一時的に大人数の予定を合わせたい人**です。

例えば、

- 大学生のクラスやゼミ
- サークル
- 研究室
- 飲み会やご飯会
- イベント運営
- グループ旅行
- ワークショップ
- 勉強会
- 同窓会

などが考えられます。

特に大学生向けは相性がよいです。

授業、バイト、サークル、研究室、遊びなどで予定が散らばっており、かつ予定内容を細かく共有したいわけではないためです。

# 主要機能

### 1. イベントページ作成

幹事がイベント名、候補期間、対象時間帯を設定します。

例：

- 5月20日から5月31日
- 10:00から22:00
- 30分単位または1時間単位

### 2. URL共有

作成したページをLINEやSNSで共有します。

参加者はアプリを入れずに、Webブラウザから参加できます。

### 3. 空き時間の入力

参加者は、自分が参加できる時間をカレンダー上で選択します。

予定の内容を書く必要はありません。

入力方法は、

- 空いている時間を塗る
- 参加できない時間を塗る
- Googleカレンダー連携で自動入力する

などが考えられます。

最初のMVPでは、手動入力だけで十分です。

### 4. 空き具合のカラーグレード表示

各時間枠に対して、何人が参加可能かを集計し、色の濃淡で表示します。

例：

- 濃い色：多くの人が参加可能
- 中間色：半分程度が参加可能
- 薄い色：参加可能人数が少ない
- グレー：ほぼ参加不可

これにより、幹事は候補時間を一目で判断できます。

### 5. 候補時間ランキング

参加可能人数が多い時間を自動でランキング表示します。

例：

1位：木曜 18:00から20:00，27人参加可能

2位：金曜 19:00から21:00，25人参加可能

3位：火曜 17:00から19:00，23人参加可能

これがあると、幹事の最終判断がかなり楽になります。

### 6. 必須メンバー設定

先生、代表者、発表者など、必ず参加してほしい人を設定できるようにします。

単純に人数が多い時間ではなく、必須メンバーが参加できる時間を優先して候補にできます。

これは大人数調整ではかなり実用的です。

# このアプリの強み

一番の強みは、**大人数の日程調整に特化していること**です。

GoogleカレンダーやTimeTreeは、継続的に予定を共有する相手には便利ですが、一時的な30人規模の調整には重いです。

一方で、調整さんやLINEスケジュールは手軽ですが、候補日を先に決める必要があります。

このアプリは、その中間にあります。

- アプリ登録なしで使える
- URL共有で参加できる
- 予定内容は見せない
- 大人数の空き具合を色で見られる
- 候補日を先に絞らなくてもよい

この組み合わせが価値です。

# このアプリの弱み・課題

### 1. 入力してもらう手間は残る → Googleカレンダーとの連携とか？？

Webで簡単に使えるとしても、参加者が予定を入力しなければ成立しません。

そのため、入力をどれだけ簡単にできるかが重要です。

例えば、

- スワイプで一気に選択
- 前日と同じ予定をコピー
- 平日夜だけ入力
- 空いている時間だけ選択
- 忙しい時間だけ選択

などが必要です。

### 2. プライバシー推測の問題

予定内容を隠しても、時間帯や埋まり方からある程度推測される可能性があります。

特に人数が少ない場合は、誰がその時間に空いていないか分かってしまう可能性があります。

対策としては、

- 一定人数が回答するまで結果を見せない
- **個人名を表示しない → 運営側だけ見えるとか？見えるかどうかを選択式にするとか？**
- 人数ではなく段階表示にする
- 回答直後にリアルタイム反映しない
- 幹事だけが詳細な集計を見られる

などが考えられます。

### 3. 既存サービスとの差別化を明確にする必要がある

単に予定調整ができます、だけだと調整さんやLINEスケジュールに近くなります。

そのため、差別化の軸は必ず、

**大人数，非公開，カラーグレード，可視化，Webリンク共有**

に置くべきです。
